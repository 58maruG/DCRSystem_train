## A. 特定クラスの枚数が少ない（クラス不均衡）

### 状況の確認方法

```bash
# data.yaml の train フォルダに含まれる各クラスのアノテーション数を数える
# labels/ フォルダ内の .txt ファイルの先頭列がクラスID
```

学習後の `results.csv` や混同行列でクラスごとの Precision / Recall を確認する。
枚数が少ないクラスだけ極端に低い場合は不均衡が原因。

---

### 対策 1: `copy_paste` 拡張を有効にする ⭐ 最優先

少数クラスのオブジェクトを別画像に貼り付けて疑似的にデータを増やす。

```python
copy_paste=0.3,   # 0.1〜0.5 で試す。0.5 以上は副作用が出やすい
mosaic=1.0,       # copy_paste は mosaic と組み合わせると効く
close_mosaic=15,  # 最後の15エポックで mosaic を切る
```

> **注意**: 背景と物体のなじみが悪いと精度が落ちることもある。混同行列を見て判断する。

---

### 対策 2: `cls` ロス重みを上げる

クラス分類ロスの重みを増やすことで、モデルに「クラスを正しく当てることが重要」と伝える。

```python
cls=1.0,   # デフォルト 0.5 → 1.0〜2.0 で試す
```

---

### 対策 3: データを追加する（根本解決）

アノテーション追加が最も確実。目安として：

| クラスの難易度 | 最低枚数の目安 |
|---------------|--------------|
| 形がシンプル・均一 | 100〜200枚 |
| バリエーションが多い | 300〜500枚 |
| 背景に紛れやすい | 500枚以上 |

---

### 対策 4: 多いクラスを間引いて比率を整える（`fraction` + カスタムサンプリング）

データ量の多いクラスを減らして比率を近づける方法。
ただし全体のデータ量が減るリスクがあるため、他の対策が難しい場合の最終手段。

---

### 試す順番

```
copy_paste を 0.3 で試す
  → 改善しない場合: cls を 1.0 に上げる
    → まだ低い場合: データ追加を検討
```

## B. 特定クラスの mAP が低い・特徴量が小さくて学習できていない不安

### 「特徴量が小さい」とはどういう状態か

- 画像内で物体が小さい（ピクセル数が少ない）
- 物体の色・形・テクスチャが背景や他クラスと似ている
- 照明・角度によって見た目が大きく変わる

---

### 対策 1: `imgsz` を大きくする ⭐ 最優先

解像度を上げることで小さい特徴も拾いやすくなる。

```python
imgsz=1280,   # 640 → 1280。VRAM が足りなければ batch を下げる
batch=20,     # imgsz=1280 の場合、VRAM 使用量が増えるので調整
```

> **確認**: `imgsz` を上げても物体サイズが画像全体に対して小さいままなら効果は限定的。

---

### 対策 2: モデルサイズを上げる

特徴量が複雑な場合、モデルの表現力が足りない可能性がある。

```python
model = YOLO("yolo11m.pt")  # s → m。さらに必要なら l
```

| 変更 | 期待できる効果 |
|------|--------------|
| s → m | 細かい特徴をより拾いやすくなる |
| m → l | さらに複雑なパターンに対応 |

---

### 対策 3: HSV 拡張を強めて照明変化に強くする

照明・色の違いに過学習している場合に有効。

```python
hsv_h=0.03,   # デフォルト 0.015 → 倍にする
hsv_s=0.9,    # デフォルト 0.7 → やや強め
hsv_v=0.5,    # デフォルト 0.4 → やや強め
```

---

### 対策 4: スケール拡張を強めて物体の大小変化に対応する

```python
scale=0.7,    # デフォルト 0.5 → 0.7。物体が様々なサイズで登場するよう学習
multi_scale=True,  # バッチごとにランダムなサイズで学習
```

---

### 対策 5: `epochs` と `patience` を増やして学習を深める

特徴量が少ない = 学習に時間がかかる。

```python
epochs=300,
patience=60,
```

---

### 対策 6: 特定クラスのみ追加で可視化・確認

学習後の確認手順：

1. `runs/detect/<name>/` 内の `val_batch*_pred.jpg` を確認する
2. 特定クラスだけ検出できていない or 他クラスと混同しているか確認
3. 混同行列（`confusion_matrix.png`）でどのクラスと混同しているか確認

---

### 試す順番

```
imgsz を 1280 に上げる（VRAM が足りる場合）
  → 改善しない場合: モデルを s → m に変更
    → さらに: epochs と patience を増やす
      → 依然として低い場合: データの質・アノテーションを見直す
```

## C. 過学習（val loss が上昇している）

### 確認方法

`results.csv` の `val/box_loss`、`val/cls_loss` が途中から上昇傾向にある場合。

### 対策

```python
weight_decay=0.001,   # デフォルト 0.0005 → 倍にする
dropout=0.1,          # 正則化（検出タスクでは控えめに）
# augmentation を強化
degrees=10,
scale=0.6,
copy_paste=0.2,
```

---

## D. 早期停止が早すぎる・学習が浅い

### 確認方法

実際の停止エポックが `patience` より大幅に小さい場合。
`results.csv` の mAP が十分上がる前に止まっている。

### 対策

```python
patience=60,          # デフォルト 40 → 60〜80
warmup_epochs=5,      # ウォームアップを長くする
lr0=0.005,            # 初期学習率を下げて安定させる
cos_lr=True,          # 学習率スケジュールを滑らかに
```
